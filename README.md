# ULTRA Agency

Локальное веб-приложение для учета моделей, заказчиков, проектов, сотрудников и аналитики модельного агентства.

Проект сделан на Django Templates + SQLite без DRF, React и SPA. Интерфейс использует Bootstrap 5 через CDN.

## Возможности

- учет анкет моделей
- хранение главного фото и дополнительных фото модели
- поиск и фильтрация моделей
- учет заказчиков
- учет проектов, кастингов, фотосессий, показов и рекламных проектов
- назначение моделей на проекты через участия
- контроль статусов моделей, проектов и участий
- кадровый учет сотрудников
- dashboard с основными метриками
- аналитические отчеты по моделям, персоналу и выручке
- роли через Django Groups + `EmployeeProfile`

## Роли

В системе предусмотрены роли:

- Директор агентства
- Букер
- Скаут
- Системный администратор

Доступы:

- Директор: просмотр базы, сотрудники, dashboard, полные отчеты и финансы.
- Букер: CRUD моделей, заказчиков и проектов, ограниченные отчеты без финансов.
- Скаут: добавление и просмотр моделей, отчеты по популярности моделей без финансов.
- Системный администратор: полный доступ.

## Быстрый старт после клонирования

Клонировать репозиторий:

```powershell
git clone <URL_РЕПОЗИТОРИЯ>
cd ULTRA
```

Создать и активировать виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

Обновить `pip`:

```powershell
python -m pip install --upgrade pip
```

Установить зависимости:

```powershell
pip install Django Pillow
```

Применить миграции:

```powershell
python manage.py migrate
```

Создать тестовые данные:

```powershell
python manage.py seed_demo_data
```

Запустить сервер:

```powershell
python manage.py runserver
```

Открыть приложение:

```text
http://127.0.0.1:8000/
```

Админка:

```text
http://127.0.0.1:8000/admin/
```

## Тестовые пользователи

После запуска команды `seed_demo_data` создаются пользователи:

```text
admin
director
booker
scout
```

Пароль по умолчанию:

```text
Ultra12345!
```

Можно задать другой пароль при создании демо-данных:

```powershell
python manage.py seed_demo_data --password MyPassword123!
```

Важно: команда не перезаписывает пароль уже существующих пользователей. Новый пароль применяется только при создании пользователя.

## User Guide

### 1. Вход в систему

Откройте:

```text
http://127.0.0.1:8000/login/
```

Введите один из тестовых логинов и пароль. Для полного доступа используйте `admin` или `director`.

### 2. Dashboard

Главная страница показывает:

- общее количество моделей
- активных моделей
- моделей в архиве
- моделей на контракте
- открытые проекты
- активных сотрудников
- выручку за текущий месяц для директора и администратора
- последние модели
- последние проекты

### 3. Модели

Раздел:

```text
/models/
```

Можно:

- смотреть список моделей
- искать по имени, фамилии, телефону и email
- фильтровать по категории, статусу, росту и наличию опыта
- сортировать по имени и дате добавления
- открывать карточку модели
- создавать, редактировать и удалять анкеты при наличии прав

Карточка модели показывает:

- основную информацию
- параметры
- статус и категорию
- главное фото
- дополнительные фото
- служебную информацию

### 4. Заказчики

Раздел:

```text
/clients/
```

Можно:

- вести базу заказчиков
- искать заказчика по названию, контактному лицу, телефону, email и городу
- смотреть карточку заказчика
- видеть проекты заказчика
- создавать, редактировать и удалять заказчиков при наличии прав

Если у заказчика есть проекты, удалить его нельзя из-за защищенной связи в базе.

### 5. Проекты

Раздел:

```text
/projects/
```

Можно:

- вести проекты, кастинги, фотосессии, показы и рекламу
- фильтровать проекты по статусу, типу, заказчику и датам
- искать по названию, описанию и заказчику
- открывать карточку проекта
- назначать моделей на проект через `Participation`
- видеть статус участия каждой модели

Статусы участия:

- приглашена
- на рассмотрении
- утверждена
- отказ
- завершено

### 6. Сотрудники

Раздел:

```text
/employees/
```

Доступен только директору и системному администратору.

Показывает:

- список сотрудников
- роль
- телефон
- email
- дату приема
- активность
- карточку сотрудника
- проекты, где сотрудник назначен ответственным

Создание и редактирование сотрудников сейчас выполняется через Django admin.

### 7. Отчеты

Раздел:

```text
/reports/
```

Можно выбрать период и посмотреть:

- популярность моделей
- количество участий модели в проектах
- количество утверждений модели
- финансовый отчет по завершенным проектам
- занятость персонала

Финансовая аналитика доступна только директору и системному администратору.

## Media и изображения

Фотографии моделей хранятся через `ImageField`.

Настройки:

```python
MEDIA_URL = 'media/'
MEDIA_ROOT = BASE_DIR / 'media'
```

В dev-режиме media раздается через `ULTRA/urls.py`, если `DEBUG=True`.

## Ограничения текущей версии

- Проект рассчитан на локальный запуск.
- Используется SQLite.
- `DEBUG=True`, настройки не готовы для production.
- Нет автоматического ресайза загружаемых изображений.
- Нет полноценного CRUD сотрудников в пользовательском интерфейсе.
- Тест-кейсы пока не написаны.
- Права доступа реализованы простыми role-based mixins.

## Перед публикацией в production

Для production нужно:

- вынести `SECRET_KEY` в переменные окружения
- выключить `DEBUG`
- настроить `ALLOWED_HOSTS`
- настроить нормальную раздачу static/media
- заменить SQLite на PostgreSQL, если появится нагрузка
- добавить тесты
- добавить резервное копирование базы и media
